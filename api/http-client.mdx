## Overview

The `httpClient` provides a unified interface for making HTTP requests to the APM Enterprise API. It handles request configuration, authentication headers, and response processing.

## Configuration

### BASE_URL

All requests are made relative to the base URL:

```javascript
const BASE_URL = "https://api.apm-enterprise.com/v1";
```

### Default Headers

Every request includes the following default headers:

<ParamField path="Content-Type" type="string" default="application/json">
  Specifies JSON content type for request body
</ParamField>

<ParamField path="Authorization" type="string" default="Bearer demo-token">
  Authentication token for API access
</ParamField>

## Methods

### get()

Make a GET request to fetch data from the API.

```javascript
httpClient.get(endpoint, options)
```

<ParamField path="endpoint" type="string" required>
  The API endpoint path (e.g., `/dashboard`, `/projects/123`)
</ParamField>

<ParamField path="options" type="object">
  Optional configuration object
  
  <Expandable title="properties">
    <ParamField path="headers" type="object">
      Additional headers to merge with defaults
    </ParamField>
  </Expandable>
</ParamField>

**Returns:** `Promise<Object>` - Parsed JSON response

#### Example

```javascript
import { httpClient } from './services/httpClient';

// Basic GET request
const data = await httpClient.get("/dashboard");

// GET with custom headers
const response = await httpClient.get("/dashboard", {
  headers: { Authorization: `Bearer ${token}` }
});
```

**Real-world usage from `src/services/api.js:4`:**

```javascript
export async function apiGetDashboard({ token }) {
  const response = await httpClient.get("/dashboard", {
    headers: { Authorization: `Bearer ${token}` }
  });
  return response;
}
```

### patch()

Make a PATCH request to partially update a resource.

```javascript
httpClient.patch(endpoint, data, options)
```

<ParamField path="endpoint" type="string" required>
  The API endpoint path (typically includes resource ID)
</ParamField>

<ParamField path="data" type="object" required>
  Partial resource data to update (can be empty object)
</ParamField>

<ParamField path="options" type="object">
  Optional configuration object
  
  <Expandable title="properties">
    <ParamField path="headers" type="object">
      Additional headers to merge with defaults
    </ParamField>
  </Expandable>
</ParamField>

**Returns:** `Promise<Object>` - Parsed JSON response

#### Example

```javascript
// Toggle project status (no data needed)
const result = await httpClient.patch(`/projects/${id}/toggle`, {});

// Partially update a project
const updated = await httpClient.patch(`/projects/${id}`, {
  status: "on-hold"
});
```

```javascript

  return httpClient.patch(`/projects/${id}/toggle`, {}, {
    headers: { Authorization: `Bearer ${token}` }
  });
}
```

## Header Customization

The `getHeaders()` helper function merges custom headers with defaults:

```javascript
const getHeaders = (extraHeaders = {}) => ({
  'Content-Type': 'application/json',
  'Authorization': 'Bearer demo-token',
  ...extraHeaders,
});
```

This allows you to override default headers by passing them in the `options.headers` object:

```javascript
// Override authorization header
await httpClient.get("/dashboard", {
  headers: { 
    Authorization: `Bearer ${userToken}`,
    'X-Custom-Header': 'value'
  }
});
```
