# User Management

The APM Enterprise User Management system provides a comprehensive directory for managing user identities, viewing detailed profiles, and auditing access permissions across your organization's network.

## User Directory Overview

The **"Directorio de Usuarios"** (User Directory) serves as the central hub for identity management with the subtitle **"Gestión de Identidades de Red"** (Network Identity Management).

### Accessing the User Directory

Navigate to the Users page to view the complete list of registered users in your organization. The directory displays users in a clean, organized table format with powerful inspection capabilities.

```jsx
// Source: UsersPage.jsx:107-122
<header>
  <div>
    <h1>Directorio de Usuarios</h1>
    <p>Gestión de Identidades de Red</p>
  </div>
  <Button onClick={fetchUsers} disabled={loading}>
    <RefreshCw /> Actualizar
  </Button>
</header>
```

### Pagination Controls

The user list implements pagination for optimal performance:

- **Page Size**: 5 users per page
- **Page Indicator**: Shows current page and total pages ("Página X de Y")
- **Navigation**: Left and right chevron buttons
- **State Management**: Buttons disable at boundaries (first/last page)

```jsx
// Source: UsersPage.jsx:42-44
const pageSize = 5;
const totalPages = Math.ceil(filtered.length / pageSize);
const paginated = filtered.slice((page - 1) * pageSize, page * pageSize);

// Pagination UI (UsersPage.jsx:203-223)
<div className="p-6 border-t flex items-center justify-between">
  <p>Página {page} de {totalPages || 1}</p>
  <div className="flex gap-2">
    <button disabled={page === 1} onClick={() => setPage(p => p - 1)}>
      <ChevronLeft size={18} />
    </button>
    <button disabled={page === totalPages} onClick={() => setPage(p => p + 1)}>
      <ChevronRight size={18} />
    </button>
  </div>
</div>
```

## Deep Audit Traceability

When you click the **"Inspeccionar"** button, APM Enterprise performs a deep network audit in the background, logging detailed traceability data to the browser console.

### Audit Process

<Steps>
  <Step title="Initiate Audit">
    Clicking "Inspeccionar" triggers the audit protocol:
    
    ```jsx
    // Source: UsersPage.jsx:169-191
    onClick={async () => {
      setSelectedUser(u);
      try {
        console.group(`🔍 Auditoría Técnica: Usuario [${u.id}] ${u.name}`);
        console.log("🚀 Iniciando trazabilidad de red profunda...");
        
        // Fetch and log user posts
        const posts = await apiClient.getUserPosts(u.id);
        console.log(`📦 Posts detectados (${posts.length}):`);
        console.table(posts.slice(0, 3));
        
        // Fetch and log post comments
        if (posts.length > 0) {
          const comments = await apiClient.getPostComments(posts[0].id);
          console.log(`💬 Comentarios en el post insignia [${posts[0].id}]:`);
          console.table(comments.slice(0, 3));
        }
        
        console.log("✅ Trazabilidad completada con éxito.");
        console.groupEnd();
      } catch (err) {
        console.warn("⚠️ Auditoría profunda limitada:", err.message);
        console.groupEnd();
      }
    }}
    ```
  </Step>
  
  <Step title="Data Collection">
    The audit collects:
    
    - **User Posts**: All content created by the user
    - **Post Comments**: Comments on the user's flagship post
    - **Activity Metrics**: Engagement and interaction data
  </Step>
  
  <Step title="Console Output">
    View detailed audit logs in the browser console:
    
    ```
    🔍 Auditoría Técnica: Usuario [1] Leanne Graham
    🚀 Iniciando trazabilidad de red profunda...
    📦 Posts detectados (10):
    [Table display of posts]
    💬 Comentarios en el post insignia [1]:
    [Table display of comments]
    ✅ Trazabilidad completada con éxito.
    ```
  </Step>
</Steps>

<Note type="info">

## Data Integration

APM Enterprise integrates with external data sources for user information:

### User Data Source

The system fetches user data from external APIs using the `useUsers` hook and `apiClient`:

```jsx
// Source: UsersPage.jsx:19-20, 27
import { useUsers } from "../hooks/useUsers";
import { apiClient } from "../services/apiClient";

const { users, loading, error, fetchUsers } = useUsers();
```

### Avatar Generation

User avatars are dynamically generated using the DiceBear API:

- **Service**: DiceBear Avataaars API v7
- **URL Pattern**: `https://api.dicebear.com/7.x/avataaars/svg?seed={username}`
- **Seed**: Based on username for consistency
- **Format**: SVG for crisp display at any size

<Note>
The avatar seed uses the username, ensuring each user has a unique but consistent avatar across sessions.
</Note>

## Error Handling

The Users page implements error handling that propagates to parent components:

```jsx
// Source: UsersPage.jsx:32-36
useEffect(() => {
  if (error && onError) {
    onError(error.status || 500, error.message || "Falla de Auditoría");
  }
}, [error, onError]);
```

Errors trigger the `onError` callback with:
- **Status Code**: HTTP error code (default 500)
- **Message**: Error description (default "Falla de Auditoría")

<Note type="warning">
Network errors during user fetching or audit operations will display appropriate error messages to maintain system transparency.
</Note>
