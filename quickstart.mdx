Get started with APM Enterprise, a professional-grade network auditing and enterprise management system built with React and Vite.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 18.0 or higher
- **npm** 8.0+ (or yarn/pnpm)
- **Git** for cloning the repository

## Quick Start

<Steps>
  <Step title="Clone the Repository">
    Clone the APM Enterprise repository to your local machine:

    ```bash
    git clone https://github.com/your-org/apm-enterprise.git
    cd apm-enterprise
    ```
  </Step>

  <Step title="Install Dependencies">
    Install the required dependencies using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    The project uses the following core dependencies:
    - **React** 18.2.0 - UI framework
    - **Vite** 5.2.0 - Build tool and dev server
    - **TailwindCSS** 4.2.1 - Utility-first CSS framework
    - **Lucide React** - Icon library
    - **Radix UI** - Accessible component primitives
  </Step>

  <Step title="Start the Development Server">
    Launch the development server with hot-reload enabled:

    ```bash
    npm run dev
    ```

    The application will start on `http://localhost:5173` by default.

    <Note>
      Vite provides lightning-fast Hot Module Replacement (HMR) for an optimal development experience.
    </Note>
  </Step>

  <Step title="Access the Application">
    Open your browser and navigate to `http://localhost:5173`. You'll see the corporate authentication screen.
  </Step>
</Steps>

## First Login

When you first access APM Enterprise, you'll be greeted with the corporate authentication screen.

The authentication screen includes:

- **Institutional Email** field for corporate login
- **Audit Key** (password) field for secure access
- Clean, professional design with the APM logo

  For development purposes, the application uses a mock authentication system. Any credentials will work to access the dashboard.

## Main Dashboard Overview

After successful authentication, you'll see the main dashboard with three key sections:

### 1. Statistics Overview

The dashboard displays three critical metrics at the top:

- **Total Revenue** - Monthly revenue tracking
- **New Users** - User growth over the last 7 days
- **Churn Rate** - User retention metrics

```jsx src/pages/DashboardPage.jsx
<section className="grid grid-cols-1 gap-6 md:grid-cols-3">
    <StatCard
        title="Ingresos Totales"
        value={dash.data ? money(dash.data.stats.revenue) : "$0.00"}
        hint="Mensual"
        icon={<DollarSign size={24} />}
    />
    <StatCard
        title="Nuevos Usuarios"
        value={dash.data ? String(dash.data.stats.newUsers) : "0"}
        hint="Últimos 7 días"
        icon={<Users size={24} />}
    />
    <StatCard
        title="Tasa de Abandono"
        value={dash.data ? `${Math.round(dash.data.stats.churn * 100)}%` : "0%"}
        hint="Retención"
        icon={<TrendingUp size={24} />}
    />
</section>
```

### 2. Project Management

The central area features a comprehensive project list with:

- Filterable status (All, Active, Paused)
- Project details including owner and budget
- Interactive project selection
- Real-time status indicators

### 3. Network Engineering Panel

The sidebar includes an API engineering control panel for simulating network conditions:

```jsx src/components/Layout.jsx
<div className="mt-4 space-y-2">
    <div className="grid grid-cols-3 gap-2">
        <button onClick={() => window.setSimulatedError(500)}>
            500
        </button>
        <button onClick={() => window.setSimulatedError(401)}>
            401
        </button>
        <button onClick={() => window.setSimulatedError(404)}>
            404
        </button>
    </div>
    <button onClick={() => window.setSimulatedError(null)}>
        Restaurar (200)
    </button>
</div>
```

<Info>
  The Network Engineering Panel allows you to simulate different HTTP error codes to test the application's error handling and resilience.
</Info>

## Available Scripts

APM Enterprise includes the following npm scripts:

<CodeGroup>
  ```bash Development
  npm run dev
  ```

  ```bash Production Build
  npm run build
  ```

  ```bash Preview Production Build
  npm run preview
  ```
</CodeGroup>

From `package.json`:
```json
"scripts": {
  "dev": "vite",
  "build": "vite build",
  "preview": "vite preview"
}
```

### Development Mode

Starts the Vite development server with HMR:
```bash
npm run dev
```

### Production Build

Creates an optimized production bundle in the `dist` directory:
```bash
npm run build
```

### Preview Production Build

Serves the production build locally for testing:
```bash
npm run preview
```
