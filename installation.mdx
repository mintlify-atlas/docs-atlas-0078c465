This guide covers the complete installation process for APM Enterprise, from system requirements to advanced configuration.

## System Requirements

Before installing APM Enterprise, ensure your system meets the following requirements:

<Info>
  **Minimum Requirements**
  - Node.js 18.0 or higher
  - npm 8.0+ (comes bundled with Node.js)
  - 2GB available RAM
  - Modern web browser (Chrome, Firefox, Safari, or Edge)
</Info>

<Tip>
  **Recommended Setup**
  - Node.js 20.x LTS
  - npm 10.x or pnpm 8.x for faster installs
  - 4GB+ RAM for optimal development experience
  - VS Code with ES7+ React/Redux/React-Native snippets extension
</Tip>

### Verify Node.js Installation

Check your current Node.js and npm versions:

```bash
node --version  # Should output v18.0.0 or higher
npm --version   # Should output 8.0.0 or higher
```

If you need to install or update Node.js, visit [nodejs.org](https://nodejs.org/) or use a version manager like [nvm](https://github.com/nvm-sh/nvm).

## Installation Methods

<Steps>
  <Step title="Clone the Repository">
    First, clone the APM Enterprise repository:

    ```bash
    git clone https://github.com/your-org/apm-enterprise.git
    cd apm-enterprise
    ```

    Or if you prefer SSH:

    ```bash
    git clone git@github.com:your-org/apm-enterprise.git
    cd apm-enterprise
    ```
  </Step>

  <Step title="Install Dependencies">
    Choose your preferred package manager and install dependencies:

      **Installation time**: Typically takes 1-3 minutes depending on your internet connection and package manager.

## Configuration Files

APM Enterprise comes pre-configured with sensible defaults. Here are the key configuration files:

### Vite Configuration

The Vite configuration is minimal and focused:

```javascript vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
})
```

<Tip>
  The default Vite configuration works out of the box. You can customize it later for path aliases, environment variables, or build optimizations.
</Tip>

### TailwindCSS Configuration

Custom color palette and theme extensions:

```javascript tailwind.config.js
export default {
```

**Custom Colors:**

- `obsidian` - Deep black for text (#1C1917)
- `accent` - Light gold highlights (#E5C07B)

## Environment Setup

APM Enterprise runs entirely on the frontend with a mock API server. No environment variables are required for basic operation.

### Optional Environment Variables

For production deployment, you may want to configure:

<CodeGroup>
  ```bash .env.local
  # API Configuration (Future Use)
  VITE_API_BASE_URL=https://api.apm-enterprise.com
  VITE_API_TIMEOUT=30000
  
  # Feature Flags
  VITE_ENABLE_ANALYTICS=true
  VITE_ENABLE_DEBUG_MODE=false
  ```

  ```bash .env.production
  # Production API Endpoint
  VITE_API_BASE_URL=https://api.apm-enterprise.com
  VITE_API_TIMEOUT=30000
  VITE_ENABLE_DEBUG_MODE=false
  ```
</CodeGroup>

<Warning>
  Never commit `.env` files containing sensitive credentials to version control. These files are already included in `.gitignore`.
</Warning>

## Troubleshooting

### Port Already in Use

If port 5173 is already in use:

```bash
# Vite will automatically try the next available port
# Or specify a custom port:
vite --port 3000
```

### Node Version Issues

If you encounter errors related to Node.js version:

```bash
# Check your Node version
node --version

# Use nvm to switch versions
nvm install 20
nvm use 20
```

### Module Not Found Errors

If you see "Module not found" errors:

```bash
# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

### Build Failures

If the build fails:

```bash
# Clear Vite cache
rm -rf node_modules/.vite

# Reinstall dependencies
npm install

# Try building again
npm run build
```
