## Atomic Component Design

APM Enterprise follows an **atomic design pattern** where components are fragmented into pure, reusable building blocks that don't handle business logic - they only render properties.

<Note>
  All components are "presentation-only" - they accept props and render UI. They don't fetch data, manage state, or contain business rules.
</Note>

## Core Components

### StatCard

Displays key metrics with icons and visual hierarchy.

**Source**: `src/components/StatCard.jsx:5-27`

<CodeGroup>
```jsx Component Definition
import React from "react";
import { Card } from "./Card";
import { cn } from "../utils/cn";

export function StatCard({ title, value, hint, icon, className }) {
    return (

                    {icon}
                </div>
                {hint && (
                    <span className="text-[10px] font-bold uppercase tracking-wider text-muted-foreground bg-muted/20 px-2 py-1 rounded-lg">

                    </span>
                )}
            </div>

            <div>
                <p className="text-sm font-medium text-muted-foreground">{title}</p>
                <h3 className="mt-1 text-3xl font-black tracking-tight text-obsidian">{value}</h3>
            </div>

            <div className="absolute -bottom-4 -right-4 h-24 w-24 opacity-[0.03] inca-pattern pointer-events-none rounded-full" />
        </Card>
    );
}
```

### Card

**Source**: `src/components/Card.jsx:4-18`

```jsx Component Definition
import React from "react";
import { cn } from "../utils/cn";

export function Card({ children, className, variant = "default" }) {
    return (
        <div
            className={cn(
```

### Badge

Status indicators with semantic color variants.

**Source**: `src/components/Badge.jsx:4-25`

```jsx Component Definition
import React from "react";
import { cn } from "../utils/cn";

export function Badge({ children, variant = "neutral", className }) {
    const variants = {
        success: "bg-green-100 text-green-700 border-green-200",
        warn: "bg-[#FFE88C]/50 text-[#CA732D] border-[#FFE88C]",
        danger: "bg-red-100 text-red-700 border-red-200",
        neutral: "bg-slate-100 text-slate-600 border-slate-200",
        primary: "bg-primary/10 text-primary border-primary/20",
    };

    return (
        <span className={cn(
            "inline-flex items-center gap-1.5 rounded-full border px-2.5 py-0.5 text-xs font-bold transition-all",
            variants[variant],
            className
        )}>
            <span className={cn(
                "h-1.5 w-1.5 rounded-full bg-current opacity-80",
            )} />
            {children}
        </span>
    );
}
```

**Usage Examples**:

### Button

Interactive button component with multiple variants and sizes.

**Source**: `src/components/Button.jsx:4-40`

<CodeGroup>
```jsx Component Definition
import React from "react";
import { cn } from "../utils/cn";

export function Button({
    children,
    variant = "primary",
    size = "md",
    className,
    icon,
    ...props
}) {
    const variants = {
        primary: "bg-primary text-white shadow-lg shadow-primary/20 hover:bg-primary/90",
        secondary: "bg-white text-obsidian border border-muted/30 hover:bg-muted/10",
        outline: "bg-transparent border-2 border-primary text-primary hover:bg-primary/5",
        ghost: "bg-transparent text-muted-foreground hover:bg-muted/30 hover:text-obsidian",
        danger: "bg-red-500 text-white shadow-lg shadow-red-500/20 hover:bg-red-600",
    };

    const sizes = {
        sm: "px-3 py-1.5 text-xs rounded-xl",
        md: "px-5 py-2.5 text-sm rounded-2xl",
        lg: "px-7 py-3.5 text-base rounded-3xl",
    };

    return (
        <button
            className={cn(
                "inline-flex items-center justify-center gap-2 font-bold transition-all active:scale-95 disabled:opacity-50 disabled:pointer-events-none",
                variants[variant],
                sizes[size],
                className
            )}
            {...props}
        >
            {icon && <span className="flex-shrink-0">{icon}</span>}
            {children}
        </button>
    );
}
```

### TextField

Form input component with label, error handling, and validation states.

**Source**: `src/components/TextField.jsx:4-35`

```jsx Component Definition
import React from "react";
import { cn } from "../utils/cn";

export function TextField({ label, value, onChange, placeholder, error, className, type = "text" }) {
    const id = `field-${label?.toLowerCase().replace(/\s+/g, '-')}`;

    return (
        <div className={cn("flex flex-col gap-2", className)}>
            {label && (
                <label htmlFor={id} className="text-sm font-medium text-muted-foreground px-1">
                    {label}
                </label>
            )}
            <input
                id={id}
                type={type}
                value={value}
                onChange={(e) => onChange(e.target.value)}
                placeholder={placeholder}
                className={cn(
                    "w-full rounded-2xl bg-white px-5 py-3 text-sm shadow-sm border-2 border-transparent transition-all outline-none",
                    "placeholder:text-muted-foreground/60",
                    "focus:border-primary/20 focus:ring-4 focus:ring-primary/10",
                    error ? "border-red-500/20 bg-red-50/50 focus:ring-red-500/10" : "hover:bg-muted/10",
                    className
                )}
            />
            {error && (
                <p className="px-2 text-xs font-bold text-red-500 animate-in fade-in slide-in-from-top-1">
                    {error}
                </p>
            )}
        </div>
    );
}
```

**Usage with Form Hook**:

```jsx Form Integration
function ProjectForm() {
    const { values, errors, onChange } = useForm(
        { name: "", budget: "0" },
        validateProject
    );

    );
}
```

## Component Benefits

<AccordionGroup>
  <Accordion title="Reusability">
    Components can be used anywhere in the application without modification. The same `Badge` component works in the dashboard, project list, and user profiles.
  </Accordion>

  <Accordion title="Testability">
    Since components are pure and don't have side effects, they're easy to test:
    ```jsx
    test('StatCard renders value correctly', () => {
      render(<StatCard title="Revenue" value="$10,000" />);
      expect(screen.getByText('$10,000')).toBeInTheDocument();
    });
    ```
  </Accordion>

  <Accordion title="Maintainability">
    Styling changes only need to be made once. Updating the `Button` component automatically updates all buttons across the app.
  </Accordion>

  <Accordion title="Consistency">
    Design system is enforced through components. Developers can't accidentally create inconsistent UI.
  </Accordion>
</AccordionGroup>

## Design System Integration

All components use the `cn` utility for conditional class merging:

```js utils/cn.js
import { clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs) {
    return twMerge(clsx(inputs));
}
```

This enables:
- Conditional styling based on props
- Tailwind class merging without conflicts
- Easy theme customization

<Tip>
  The `cn` utility prevents Tailwind class conflicts. For example, `cn("p-4", "p-6")` correctly results in `"p-6"` instead of both classes being applied.
</Tip>
